Assignment Description

This assignment consists of four parts.

Part 1 Part 2 Part 3 Part 4

Part 1

Create a web application with ASP.NET in C# that uses a persistent cookie to track how many times a client computer has visited the page. In addition to the number of hits, the application also needs to display the IP address of the client computer and the time zone where the client computer is located.

Part 2

Develop a web application using ASP.NET in C# that runs a slideshow of a list of pictures. The requirements are as follows:

  1. Information about the pictures must be stored on the server either in a database or in a text file. These details would include the name and location or URL of each picture and a description of the picture to be used as a caption, which must be stored in a plain text file or a database.
  2. The caption should be shown under each picture in the show.
  3. There should be a button to start/stop the show.
  4. There should be a control to toggle the show between random and sequential.
  5. There should be buttons to manually turn the show backward or forward, only if the show is in sequential mode.
  6. You may reuse the pictures you have used for assignment 1.
  7. The show should have at least 20 nice pictures.

Part 3

In parts 3 and 4 of this assignment, you will be developing an online computer store in which customers can select a computer from a list and customize it by replacing some components in its standard configuration with others that may be better or cheaper. Customers can then place an order with the online store.

In this part, your focus should be on the web interface design and implementation of the online store, using ASP.NET in C# as server-side language and other web technologies you have learned from Units 1–4. The requirements are as follows:

  1. Visit some online stores such as Bestbuy.CA and Amazon.CA, study their designs and functionalities, and write your findings in the assignment report.
  2. Based on your study above, analyze the requirements of the web application you are to develop and then work out a design of the web application, including drawings of the interface, name and functions of each modules you will need, and web pages/documents to be used.
  3. The application running in a web browser should be like a real online store, with a name and logo, menus and buttons for navigation, a welcome page to greet customers, and other pages, such as contacts and a feedback form, and so on.
  4. There should be a list of computers, lists of different parts or components (no less than five different components including at least RAM, hard drive, CPU, display, OS, and soundcard) of computers that can be reviewed by customers. At this stage, the information about these products can be hard coded into the HTML or program files.
  5. A customer can select a computer with default/standard configuration and pricing, and then customize it by changing its configuration, that is, by replacing one component with another from a list of compatible components.
  6. Whenever a change has been made by the customer, the price total needs to be dynamically changed and displayed to the customer, in an area embedded in the online store, not in a pop-up window.
  7. After the customer has hit the Buy or Add to Cart or something similar, a summary of selections, including all the details of the customized computer and its pricing, should be displayed in the browser window.
  8. At this stage, you don’t need to save orders placed by customers in the database. You can use Cookies to store and track customer selections.
  9. All links, menus, and buttons within the online store should be active.

Part 4

In part 4 of the assignment, you will continue your work completed in part 3, to develop a fully functioning online computer store (without payment transaction, though). Your tasks for this part are:

  1. Add a customer management module to the online computer store, which should include new customer registration, customer authentication, and password recovery.
  2. Add an order management module which will save orders for registered customers and allow registered customers to manage their orders, including view the list and details of each item in the list and delete or make changes to a selected order in the list.
  3. Design and implement a back-end database to support your online computer store that does the following.
    1. Store all the details of the computers and components and replace all the data hard coded into the program files with data from the database.
    2. Use the database for all your customer management.
    3. Use the database for all your order management. When an order is submitted, the details of the order, including pricing, need to be saved in the database, and then displayed in an appropriate format to the customer.

Part 1 and 2

Part 1 and 2 of this project was designed and completed using the agile/scrum development methodology. The agile/scrum development methodology is a project management system that relies on incremental development. This incremental development is achieved by dividing the work into cycles of short work periods called sprints. Each sprint consists of planning, analysis, coding and review. Each sprint time length is designated as one day long.

The analysis of Part 1 has resulted in the following requirements:

The web page must:

  • Use a persistent cookie.
  • Track how many times a client computer visits the web page.
  • Display the number of times a client has visited.
  • Display the client’s computer IP address.
  • Display the time zone of the client computer.

The analysis of Part 2 has resulted in the following requirements:

The web page must:

  • Be a slideshow.
  • Store information about the pictures in a database or text file.
  • Display a caption under each picture.
  • Have a button to start/stop the show.
  • Have a button that toggles the random and sequential transition between pictures.
  • Have button that manually turns the show backward or forward.
  • Not allow manual transition when the show is not in sequential mode.
  • Have at least 20 pictures.

The database/text file:

  • Should include the name and location of each picture.
  • Must include a description for each picture.

Part 3

Part 3 of the project was designed and completed using the waterfall development methodology. The waterfall development methodology is a project management system that breaks down the project activities into linear sequential phases. Each of these phases depends on the deliverables of the previous phase. These phases are:

  1. Requirements
  2. Design
  3. Implementation
  4. Verification
  5. Maintenance.

Purpose

The purpose of Part 3 is the development of an online computer store. Customers can select a computer from a list and customize it. Computers are customized by replacing its components in its standard configuration with other components. Customers can then place an order with the online store.

Design

The design for Part 3 began with analysing other online stores such as Bestbuy, Amazon, Staples and The Source. This analysis has revealed the following:

  • The topmost bar contains the logo, search bar, user account and cart link.
  • Logo for the website is in the upper left corner or at the top of the page in the center.
  • The logo is a link back to the main page of the store.
  • Top right contains the links to user account and cart.
  • The navigation bar is located underneath the topmost bar.
  • The navigation bar contains links or dropdown menus to different sections of the store.
  • Clicking on the links opens to a new page.
  • Underneath the topmost bar and the navigation bar is the main area or store front.
  • The main area is usually accented by two blank columns on either sides that draws the customers’ attention towards the center of the page.
  • The topmost section of the main area is dominated by any special promotion or service that the store has going on.
  • Clicking on the special promotion either loads the product or shifts to the next special promotion without reloading page.
  • Underneath the promotion is the list of services that the store has available or any products that are also on sale.
  • Bottom of the page either in the bottom left or right are links to contact information, feedback, return policy, hiring policy, or other helpful information.
  • Main color scheme of the stores are usually neutral colors (white, gray, pale blue, etc.) which are easy on the eyes and gives the feeling of nonaggressive attitude.
  • Any bright colors (ex. Red) are usually used to bring attention to specific parts of the website.
  • Clicking on add to card updates the cart after reloading page with a pop-up indicating that product was successfully added to cart.
  • Remove item from the cart, click on trash can icon next to the product.
  • Clicking on product goes to product description page.
  • When viewing cart, items are typically shown on the left while the order total is shown on the right.
  • Clicking on the checkout button on the cart page requires the user to be logged in.
